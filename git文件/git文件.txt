我们来说一下我们在拉取或者推送文件到git仓库是，

会有一个“.git文件夹”，我们在git init时也会有一个“.git文件夹”。

那么这个“.git文件夹”是干嘛使用的呢？

我们知道在我们安装git软件时，目的就是要通过该git软件能够在bush环境下通过git init

创建一个本地仓库，然后把这个仓库里所有的资源推送到GitHub远程仓库中，

因为GitHub远程仓库也是这个bush环境，如果环境不一样就会造成跨平台问题。

OK 我们目的是说“.git文件夹”，“.git文件夹”这个是在git init的时候在本地所创建的本地仓库。

如果我们要把本地仓库的资源推送到远程仓库时就需要git init一下。目的就是创建仓库，

这个文件里包含了本地仓库所有添加到本地仓库的资源信息，如果把“.git文件夹”删了，

那么该本地仓库就不存在了，如果是从远程仓库拉取的，那么本地仓库所包含的git信息也就不存在了。

也就是说本地仓库与远程仓库所拉取的文件的“.git文件夹”里所包含的信息基本相同，

不同的是git 链接资源地址不一样。你本地所创建的“.git文件夹”链接资源地址指向本地仓库连接，

你远程所创建的“.git文件夹”链接资源地址指向远程仓库连接。
